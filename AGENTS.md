# Repository Guidelines

## Overview

GTK4/libadwaita Flatpak app (`io.github.astrovm.AdventureMods`) in Rust. Automates Sonic Adventure DX and SA2 mod setup on Linux with Steam/Proton through a GUI wizard.

## Structure

```
src/
  ui/           GTK/libadwaita widgets and pages
  setup/        Setup flow, game-specific logic, mod definitions
    common.rs   Shared ModEntry, ModSource, install pipeline
    sadx.rs     29 SADX mods, presets, Steam-to-2004 conversion
    sa2.rs      13 SA2 mods
    steps.rs    Step sequencing per game
    config.rs   Shared config file generation (Default.json, Profiles.json)
    sadx_config.rs / sa2_config.rs   Game-specific config generation
  steam/        VDF parser, library folder detection
  external/     Downloads, archives, protontricks, Flatpak sandbox
data/           Desktop file, metainfo, icon, GSettings schema, UI templates, images
build-aux/      Flatpak manifests, cargo-build.sh, cargo-sources.json
po/             Translations
```

`src/config.rs` is generated by Meson from `src/config.rs.in` — edit the template, not the output.

## Build & Test

System deps: `libgtk-4-dev`, `libadwaita-1-dev`

```bash
PKG_CONFIG_PATH=/usr/lib/x86_64-linux-gnu/pkgconfig:/usr/share/pkgconfig cargo build
PKG_CONFIG_PATH=/usr/lib/x86_64-linux-gnu/pkgconfig:/usr/share/pkgconfig cargo test
cargo fmt --all
cargo clippy --all-targets

# Flatpak
flatpak-builder --user --install build build-aux/io.github.astrovm.AdventureMods.json

# After updating Cargo.lock
flatpak-cargo-generator Cargo.lock -o build-aux/cargo-sources.json
```

## Architecture

**Build system**: Meson outer (GResources, schemas, desktop files, i18n) → Cargo inner via `build-aux/cargo-build.sh`. `config.rs.in` injects APP_ID, VERSION, PKGDATADIR; `config.rs` has fallback defaults for standalone `cargo build`.

**Async**: GLib runtime, no tokio. `glib::spawn_future_local()` for UI-safe async, `gio::spawn_blocking()` for blocking work, `async-channel` for progress updates.

**Widgets**: gtk-rs composite template pattern — `mod imp {}` with `#[glib::object_subclass]`, `#[template_child]` bindings to `.ui` files, public type via `glib::wrapper!`. Templates in `data/resources/ui/`.

**Navigation**: `AdwNavigationView` stack: Welcome Page → Setup Page. Setup is a multi-step wizard from `steps_for_game()` → `Vec<SetupStep>`. Each step has a `StepKind` (Auto, Info, ExternalAction, Download, ModSelection).

**Sandbox**: `external/flatpak.rs` detects `/.flatpak-info` and routes host commands through `flatpak-spawn --host`. Foundation for protontricks and Flatpak install.

**Steam detection**: Custom VDF parser (`steam/vdf.rs`) reads `libraryfolders.vdf` to find SADX (71250) and SA2 (213610) across library folders.

**Mod pipeline**: `ModEntry` with `ModSource` (`GameBanana { file_id }` | `DirectUrl { url }`). Streaming downloads with progress via reqwest. Sources: GameBanana, dcmods.unreliable.network, GitHub, GitLab. Config generation writes `Default.json` with native resolution, optimal settings, and mod load order.

## Conventions

- **Style**: Default `rustfmt`, `snake_case` functions/modules, `PascalCase` types, `SCREAMING_SNAKE_CASE` constants. Game-specific logic stays in `sadx*`/`sa2*` files.
- **Error handling**: `anyhow::Result` everywhere, errors shown in UI, never panic.
- **Tests**: Colocated in `#[cfg(test)] mod tests`. Deterministic unit tests for config generation, VDF parsing, step sequencing. Run `cargo test` before PRs.
- **Commits**: Conventional Commits (`feat:`, `fix:`, `refactor:`, `chore:`). Imperative, specific subjects.
- **IDs**: Release `io.github.astrovm.AdventureMods`, dev `.Devel`. GResource prefix `/io/github/astrovm/AdventureMods/`. Protontricks `com.github.Matoking.protontricks`.
- **Security**: Don't commit `target/`, `build/`, extraction dirs. Validate download URLs and archive handling carefully.
