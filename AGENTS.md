# Repository Guidelines

## Overview

GTK4/libadwaita Flatpak app (`io.github.astrovm.AdventureMods`) in Rust. Automates Sonic Adventure DX and SA2 mod setup on Linux with Steam/Proton through a GUI wizard.

## Structure

```
src/
  main.rs             Entry point
  application.rs      GtkApplication subclass
  window.rs           Main AdwApplicationWindow
  config.rs.in        Build-time constants template (Meson generates config.rs)
  ui/
    welcome_page.rs   Game detection and selection
    setup_page.rs     Multi-step wizard controller
    progress_page.rs  Download/install progress display
    game_card.rs      Per-game card widget
  setup/
    common.rs         ModEntry, ModSource, install pipeline
    steps.rs          Step sequencing per game (StepKind variants)
    config.rs         Shared config generation (Default.json, Profiles.json)
    sadx.rs           29 SADX mods, presets, Steam-to-2004 conversion
    sadx_config.rs    SADX-specific config generation
    sa2.rs            13 SA2 mods
    sa2_config.rs     SA2-specific config generation
  steam/
    vdf.rs            Custom VDF parser
    library.rs        Steam library folder detection
    game.rs           Game detection (app IDs 71250, 213610)
  external/
    download.rs       Streaming downloads with progress (reqwest)
    archive.rs        Archive extraction
    protontricks.rs   Protontricks integration
    flatpak.rs        Flatpak sandbox detection and flatpak-spawn routing
data/                 Desktop file, metainfo, icon, GSettings schema, UI templates, screenshots
build-aux/            Flatpak manifests, cargo-build.sh, cargo-sources.json
po/                   Translations
```

`src/config.rs` is generated by Meson from `src/config.rs.in`. Edit the template, not the output.

## Build & Test

System deps: `libgtk-4-dev`, `libadwaita-1-dev`

```bash
PKG_CONFIG_PATH=/usr/lib/x86_64-linux-gnu/pkgconfig:/usr/share/pkgconfig cargo build
PKG_CONFIG_PATH=/usr/lib/x86_64-linux-gnu/pkgconfig:/usr/share/pkgconfig cargo test
cargo fmt --all
cargo clippy --all-targets

# Flatpak
flatpak-builder --user --install build build-aux/io.github.astrovm.AdventureMods.json

# After updating Cargo.lock
flatpak-cargo-generator Cargo.lock -o build-aux/cargo-sources.json
```

## Architecture

**Build system**: Meson wraps Cargo via `build-aux/cargo-build.sh`. Meson handles GResources, schemas, desktop files, and i18n. `config.rs.in` injects APP_ID, VERSION, PKGDATADIR at build time. `config.rs` has fallback defaults so standalone `cargo build` works too.

**Async**: GLib runtime, not tokio. `glib::spawn_future_local()` for UI-safe async, `gio::spawn_blocking()` for blocking work, `async-channel` for progress updates from background tasks to the UI.

**Widgets**: gtk-rs composite template pattern. Each widget has a `mod imp {}` with `#[glib::object_subclass]` and `#[template_child]` bindings to `.ui` XML files in `data/resources/ui/`. The public type is exposed with `glib::wrapper!`.

**Navigation**: `AdwNavigationView` with two main pages: Welcome Page (game detection/selection) and Setup Page (wizard). The wizard runs through `steps_for_game()` which returns `Vec<SetupStep>`, where each step has a `StepKind` (Auto, Info, ExternalAction, Download, ModSelection).

**Sandbox**: `external/flatpak.rs` checks for `/.flatpak-info` and routes host commands through `flatpak-spawn --host`. This is how protontricks and Flatpak install work from inside the sandbox.

**Steam detection**: Custom VDF parser (`steam/vdf.rs`) reads `libraryfolders.vdf` to locate SADX (71250) and SA2 (213610) across all library folders.

**Mod pipeline**: `ModEntry` defines each mod with a `ModSource` (`GameBanana { file_id }` or `DirectUrl { url }`). Downloads stream with progress via reqwest. Sources include GameBanana, dcmods.unreliable.network, GitHub, and GitLab. Config generation writes `Default.json` with native resolution, optimal settings, and mod load order.

## Conventions

- **Writing style**: Text must sound natural and human-written. Avoid unnecessary em-dashes (use commas, periods, or nothing), overly complex sentences, AI-typical phrasing, corporate or excessively formal language, and bullet lists where a paragraph works better. Keep the tone conversational, direct, and simple.
- **Rust style**: Default `rustfmt`, `snake_case` for functions/modules, `PascalCase` for types, `SCREAMING_SNAKE_CASE` for constants. Game-specific logic stays in `sadx*`/`sa2*` files.
- **Error handling**: `anyhow::Result` everywhere. Errors get shown in the UI. Never panic.
- **Tests**: Colocated in `#[cfg(test)] mod tests`. Deterministic unit tests for config generation, VDF parsing, step sequencing. Run `cargo test` before PRs.
- **Commits**: Conventional Commits (`feat:`, `fix:`, `refactor:`, `chore:`). Imperative, specific subjects.
- **IDs**: Release `io.github.astrovm.AdventureMods`, dev `.Devel`. GResource prefix `/io/github/astrovm/AdventureMods/`. Protontricks `com.github.Matoking.protontricks`.
- **Security**: Don't commit `target/`, `build/`, or extraction dirs. Validate download URLs and archive handling carefully.
